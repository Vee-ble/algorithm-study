### **1. 요구사항 분석**

**요구사항 요약: 컴퓨터의 수와 컴퓨터들간의 연결이 주어질 때 1번 컴퓨터가 웜 바이러스에 걸려서 인접한 컴퓨터에 전파된다. 1번을 통해 전파되는 컴퓨터의 수를 구하라.**

**제약 사항(Constraints): 컴퓨터의 수 ≤ 100**

**예외 케이스(Edge Case):**

- 컴퓨터의 수가 1개이거나 쌍이 1개일 때?

### **2. 해결 전략 및 자료구조 설계**

**해결 전략 (Algorithm):**

- 연결만큼 loop 하고, key(왼쪽 번호): [value,value](오른쪽 번호)로 저장한다.
- 큐에 1을 넣고, 1이 가진 value들을 큐에 넣는다.
- 해당 큐에서 다시 다음 연결을 가져와서 이를 반복한다.

**선택한 자료구조 근거:**

- 선택: key, value 형태로 저장해두는 건 map, queue는 배열로 구현.
- 이유: has로 쉽게 접근할 수 있고, queue는 배열의 shift와 push로 간단하게 구현가능하다. 하지만 성능을 고려한다면 직접 구현하거나, index로 dequeue를 구현한다.

**Trade-off 분석:**

**예상 시간/공간 복잡도: O(N), O(N) N = 컴퓨터의 수**

### 3. 의사코드 & 검증

```jsx
입력을 loop하면서 connectMap을 구현
queue 배열에 1 삽입
queue를 while loop
	맨 앞 요소를 가져온다.
	count를 증가시킨다.
	connectMap의 value로 가지고 있는 요소들을 큐에 넣는다.

```
